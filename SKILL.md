# 紫苏叶供应链咽喉分析

## 目标

将一个股票代码、公司或供应链主题拆解成清晰的咽喉分析。重点不是追逐最显眼的“卖铲人”，而是从 AI / 半导体建设链条反向推导，寻找那些规模不大、替代困难、却可能限制巨大下游资本开支推进的窄节点。

所有输出都应定位为研究辅助，而不是金融建议。涉及上市公司事实、近期事件、价格、公告、新闻、管理层表述、股权结构和市场数据时，必须使用最新来源核验并标注出处。

## 快速流程

1. 定义目标：股票代码、公司、产品线、供应链角色、地域和研究周期。
2. 反向映射下游需求：云厂商资本开支、加速器 / 网络架构、BOM、制造流程、模块供应商、材料、设备和物流。
3. 识别咽喉：供应集中度、近似唯一来源、客户认证壁垒、产能扩张周期、替代难度、定价权和地缘脆弱性。
4. 压力测试：证据质量、替代供应商、垂直整合、技术路线变化、稀释、客户节奏和宏观 / 地缘风险。
5. 输出研究简报：评分、证据表、供应链图、催化剂、监控清单、风险和“什么证据会改变结论”。

按需读取参考资料：

- 需要打分或解释分数时，读取 `references/scoring-rubric.md`。
- 需要撰写完整备忘录时，读取 `references/report-template.md`。
- 使用网页、公告、PDF、社交媒体或不确定供应链说法前，读取 `references/source-hygiene.md`。
- 讨论相关开源项目、集成方式、建库规划或产品化路线时，读取 `references/ecosystem-radar.md`。
- 需要生成 Markdown 分析草稿时，运行 `scripts/create_analysis_brief.py`。

## 默认输出结构

多数请求使用以下结构：

- **摘要**：3-5 条，说明核心观点、不确定性和下一步证据。
- **供应链位置**：公司 / 组件位于哪里，哪些下游支出依赖它。
- **咽喉评分**：0-100 分，列出分项分数并解释最高权重驱动因素。
- **证据表**：主张、来源、置信度、新鲜度和矛盾检查。
- **催化剂时间线**：近期、6-18 个月、2027 年以后和失败触发条件。
- **稀释与资本结构**：好稀释、坏稀释、流动性、扩产用途和股东一致性。
- **反方观点**：这个论点可能失败的可信路径。
- **监控计划**：公告、财报关键词、客户认证信号、资本开支指引变化、供货协议和机构 / 内部人行为。

用户没有要求深挖时，第一版保持简洁。需要表达依赖关系时，优先使用 Mermaid 图。

## 咽喉判断镜头

优先寻找具备这些特征的节点：

- 小型上游市场控制大型下游建设。
- 合格供应商少、客户认证周期长，或存在近似唯一来源迹象。
- 替代需要重新设计、重新认证、新材料工艺或多年产能建设。
- 需求由云厂商资本开支、AI 集群网络、CPO/ELS、HBM、先进封装、功率、散热、机器人或战略材料拉动。
- 公司通过产能、良率、工艺 know-how、知识产权或客户锁定获取价值，而不是只靠叙事。
- 市场低估了时间点、经营杠杆、定价权或稀缺性。

看到这些信号要谨慎：

- 所谓“瓶颈”只是有大量替代品的大宗投入品。
- thesis 依赖未经核验的社交媒体说法或单一管理层宣传。
- 客户可能很快双供或多供。
- 公司在需求兑现前需要反复融资。
- 大型垂直整合玩家有能力把这个节点内部化。
- 技术路线可能绕开该组件。

## 分析规则

- 区分结构和催化剂：公司可以结构上重要，但短期设置并不好。
- 区分“好稀释”和“坏稀释”：用于扩产、增厚型并购或客户支持产能的融资可能是正面的；生存型融资、不透明 ATM、面向散户的掠夺式融资则很危险。
- 明确标注不确定性。未知就是未知，不要用自信猜测补洞。
- 优先使用一手证据：公告、财报电话会、投资者材料、专利、标准组织、客户引用、可靠行业研究和技术论文。
- 跟踪矛盾。如果来源在市占率、供应商身份或时间点上冲突，要直接写出冲突。
- 不要在没有来源支持时假设具名客户关系。
- 默认不输出买卖建议。若用户要求交易规划，只做情景分析和风险框架。
